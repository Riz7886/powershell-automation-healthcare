PYX HEALTH — DATABRICKS SERVERLESS DEPLOYMENT GUIDE

New Architecture — West US 2 / West US — Automated Terraform Deployment
Prepared by Syed Rizvi  |  Infrastructure Engineering  |  April 2026
IMPORTANT:  This is a NEW deployment — not a migration. Existing workspaces (pyx-warehouse-prod, pyxlake-databricks) on sub-corp-prod-001 remain completely untouched.
1 Executive Summary

This guide details the deployment of a brand-new Databricks Serverless architecture for PYX Health. This is not a migration — we are creating entirely new infrastructure in West US 2 (primary) and West US (DR) using three new Azure subscriptions arranged in a hub-spoke topology. Two fully-automated Terraform deployment options are provided: Budget (~$5,800/mo, optimized for cost with NSG-based security) and Enterprise (full security lockdown with Azure Firewall, Private Link, CMK encryption, and Sentinel SIEM). Both options leverage Databricks Serverless Compute, Unity Catalog for data governance, and are ready for HITRUST certification (Databricks Serverless HITRUST compliance available April 24, 2026). The entire deployment is automated end-to-end via Terraform — zero manual configuration required. Infrastructure deploys in approximately 30 minutes.

4 Budget vs Enterprise Comparison

Budget Deploy

~$5,800/mo | $69,600/yr
  • 6 Terraform modules
  • NSG-based network security (no Firewall)
  • No Azure Bastion (SSH/RDP via VPN)
  • Standard Key Vault (PROD)
  • Microsoft-managed encryption
  • No Sentinel SIEM
  • Spot VMs for DEV (80% savings)
  • Private Endpoints for PROD workspace + storage
  • Log Analytics 365-day (HIPAA)
  • Best for: Teams starting Databricks, cost-sensitive

Enterprise Deploy

~$7,200/mo | $86,400/yr
  • 8 Terraform modules (+ unity_catalog, phi_pipeline)
  • Azure Firewall Standard (all egress filtered)
  • Azure Bastion Standard (secure admin access)
  • Premium Key Vault (HSM-backed CMK)
  • Customer-managed keys (90-day auto-rotation)
  • Sentinel SIEM (threat detection)
  • Photon runtime for PROD ETL
  • Private Link for ALL services (Databricks, ADLS, KV)
  • PHI Pipeline (daily de-identification + validation)
  • Best for: HITRUST audit, full PHI processing

7 Step-by-Step Deployment — Budget

1 Get the Terraform Code

Download or clone the PYX-Databricks-Budget-Deploy folder. It contains:

Directory Structure
PYX-Databricks-Budget-Deploy/
  main.tf                 # Root module - wires all 6 modules
  variables.tf            # All input variables with defaults
  terraform.tfvars        # YOUR values (edit this)
  outputs.tf              # Deployment outputs
  deploy.ps1              # One-click deployment script
  modules/
    network/main.tf       # Hub-spoke VNets, subnets, NSGs, peering
    storage/main.tf       # ADLS Gen2 (DEV LRS, PROD GRS), Key Vault
    databricks_dev/main.tf    # DEV workspace, Spot cluster, SQL warehouse
    databricks_prod/main.tf   # PROD workspace, ETL cluster, Private Endpoints
    security/main.tf      # 6 AD groups, 3 SPs, RBAC, Azure Policy
    monitoring/main.tf    # Log Analytics, alerts, cost budgets

3 Deploy (Automated Script)

PowerShell
# Navigate to the project
cd PYX-Databricks-Budget-Deploy

# Option A: Full automated deployment (recommended)
.\deploy.ps1

# Option B: Plan-only (review before apply)
.\deploy.ps1 -PlanOnly

# Option C: Manual Terraform commands
terraform init -upgrade
terraform validate
terraform plan -var-file="terraform.tfvars" -out="budget.plan"
terraform apply "budget.plan"

The deploy.ps1 script executes 7 steps automatically: prerequisite check, state backend creation, terraform init, validate, plan, apply, and output display.
