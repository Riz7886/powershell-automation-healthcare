1. Prereqs (one-time on any jump server)

STEP 1 Open PowerShell as Administrator on your jump server
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
Install-PackageProvider -Name NuGet -Force -Scope CurrentUser
Install-Module PowerShellGet -Force -Scope CurrentUser -AllowClobber
Install-Module Az.Accounts,Az.KeyVault,Az.Compute -Scope CurrentUser -Force -AllowClobber
Install-Module AWS.Tools.EC2,AWS.Tools.SimpleSystemsManagement -Scope CurrentUser -Force -AllowClobber
STEP 2 Copy the scripts to the jump server (pick one)
# Option A - copy from a network share
robocopy \\fileshare\tools\SAP-Cert-Manager C:\Tools\SAP-Cert-Manager /MIR
robocopy \\fileshare\tools\AutoHeal C:\Tools\AutoHeal /MIR
cd C:\Tools\SAP-Cert-Manager

# Option B - scripts already on your own machine
$env:Path = 'C:\Users\PC\Downloads\SAP-Cert-Manager;C:\Users\PC\Downloads\AutoHeal;' + $env:Path

# Verify either way
Get-ChildItem .\Update-ServerCert.ps1
Get-ChildItem .\examples\Renew-Ns2otpapp.ps1
STEP 3 Get your credential object once per session
$cred = Get-Credential   # enter DOMAIN\username + password
# or for local admin
$cred = Get-Credential -UserName '.\Administrator'
STEP 4 Prove connectivity to a target before running anything heavy
Test-WSMan -ComputerName ns2otpapp -Credential $cred -Authentication Default
# If that fails, try:
Test-WSMan -ComputerName 10.168.0.32 -Credential $cred -Authentication Default
# Confirm you can open a session
Enter-PSSession -ComputerName ns2otpapp -Credential $cred
exit

2. Tomorrow 4/26/2026 - ns2otpapp Cert Renewal

What's happening: Cert CN=ns2otpapp.sap.parker.corp expires 4/26/2026. Parker Hannifin General Issuing CA v2 issues the replacement. Script runs from your jump server over WinRM.
STEP 1 Open PowerShell, set credential
cd C:\Users\PC\Downloads\SAP-Cert-Manager
$cred = Get-Credential
STEP 2 Discover what's on the server (zero changes)
.\examples\Renew-Ns2otpapp.ps1 -Credential $cred -DiscoverOnly

You'll see a table like:

Index Host        Source       Status    DaysRemaining  Subject                         Thumbprint
----- ----------- -----------  --------- -------------  -------------------------------- --------------
1     ns2otpapp   WindowsStore EXPIRING  3              CN=ns2otpapp.sap.parker.corp     AABBCC...
2     ns2otpapp   IisBinding   EXPIRING  3              CN=ns2otpapp.sap.parker.corp     AABBCC...
STEP 3 Pick the fastest path based on what Step 2 shows

Path A: Cert lives in Windows Store

.\examples\Renew-Ns2otpapp.ps1 -Credential $cred

Path B: Cert is a SAP PSE file (STRUST)

.\examples\Renew-Ns2otpapp.ps1 -Credential $cred -SapPsePin ''
# OR prompt for PSE pin
.\examples\Renew-Ns2otpapp.ps1 -Credential $cred -PromptForPasswords

Path C: Cert is a PFX file on disk

.\examples\Renew-Ns2otpapp.ps1 -Credential $cred -PfxPassword ''
# OR prompt for PFX password
.\examples\Renew-Ns2otpapp.ps1 -Credential $cred -PromptForPasswords
STEP 4 Verify after the renewal
.\Update-ServerCert.ps1 -Target ns2otpapp -Credential $cred -Action Verify -Scope All
If anything fails, run .\Update-ServerCert.ps1 -Target ns2otpapp -Credential $cred -Action Rollback and call your PKI team.

3. Cert Manager - Every Command

3.1 Discovery

# Everything on a host
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Discover -Scope All

# Windows stores only
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Discover -Scope WindowsStore

# IIS bindings
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Discover -Scope IisBinding

# PFX files on disk (scans C/D/E/F drives, common cert paths)
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Discover -Scope PfxFile -PfxPassword 'p@ss'

# SAP PSE files (E:\OTC, usr/sap, Program Files\SAP, etc.)
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Discover -Scope SapPse -SapPsePin 'pin'

# Azure Key Vault certs
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Discover -Scope AzureKv -KeyVaultName myvault

# Multiple targets at once
.\Update-ServerCert.ps1 -Target server01,server02,10.0.0.5 -Credential $cred -Action Discover -Scope All

3.2 Renewal

# By subject substring
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew -Scope WindowsStore `
    -SubjectLike 'www.example.com' -ServiceName 'W3SVC'

# By exact thumbprint
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew -Scope WindowsStore `
    -Thumbprint AABBCC112233...  -ServiceName 'W3SVC'

# Any cert expiring within 45 days
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action RenewAuto `
    -ExpiryThresholdDays 45 -NonInteractive -Force

# Override subject + SANs
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew -Scope WindowsStore `
    -Thumbprint AABBCC... `
    -SubjectOverride 'CN=api.example.com,O=Example,C=US' `
    -DnsNames 'api.example.com','api-alt.example.com' `
    -KeySize 2048

# SAP PSE renewal
.\Update-ServerCert.ps1 -Target sapserver -Credential $cred -Action Renew -Scope SapPse `
    -SapPsePath 'E:\usr\sap\PP4\D00\sec\SAPSSLS.pse' `
    -SapGenPsePath 'E:\usr\sap\PP4\SYS\exe\uc\NTAMD64\sapgenpse.exe' `
    -SapPsePin 'pin' `
    -ServiceName 'SAPPP4_00'

# PFX file renewal
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew -Scope PfxFile `
    -PfxPath 'E:\OTC\OpenText\cert.pfx' -PfxPassword 'p@ss'

# Force a specific CA instead of AD auto-discovery
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew `
    -Thumbprint AABBCC... `
    -CaConfig 'ca01.parker.corp\Parker Hannifin General Issuing CA v2' `
    -CaTemplate WebServer

# Fleet-wide nightly renewal of expiring certs
.\Update-ServerCert.ps1 -Target (Get-Content .\server-list.txt) -Credential $cred `
    -Action RenewAuto -Scope WindowsStore -ExpiryThresholdDays 30 `
    -NonInteractive -Force -SkipRestart

3.3 Verification / Rollback

# Chain-validate all certs on a host
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Verify -Scope All

# Show the latest backup set (does not restore automatically)
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Rollback

3.4 Dry-run

# Plan only, no changes
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew `
    -Thumbprint AABBCC... -DryRun

.\Invoke-AutoHeal.ps1 -Target (Get-Content .\servers.txt) -Credential $cred -Mode Heal -ApplyFixes `
    -Severity Critical,High -AutoApprove -NonInteractive

.\Invoke-AutoHeal.ps1 -Target server01 -Credential $cred -Mode Rollback

5. Locked passwords & rpmsg attachments

5.1 Opening a message_v2.rpmsg (Rights Managed Email)

STEP 1 Do NOT rely on Outlook's preview pane - rpmsg cannot be previewed.
STEP 2 Double-click the attachment - Outlook opens it in a new window if your signed-in identity is authorized.
STEP 3 If blocked: go to https://outlook.office.com, find the same email, double-click there. OWA handles rpmsg natively.
STEP 4 Still blocked: install Azure Information Protection Viewer.
STEP 5 Still blocked: reply to the sender asking them to grant access to your exact email, or forward without "Do Not Forward".

5.2 Five ways to supply the password to the cert script

Option 1 - Interactive prompt

.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew `
    -Scope PfxFile -PfxPath 'E:\cert.pfx' -PromptForPasswords

Option 2 - Save once, DPAPI-encrypted, reuse forever

# First run: prompts then saves
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew `
    -Scope PfxFile -PfxPath 'E:\cert.pfx' -PromptForPasswords -SavePasswords
# Subsequent runs: no prompt needed
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew `
    -Scope PfxFile -PfxPath 'E:\cert.pfx'
# Delete a saved secret
Remove-Item "$HOME\.cert-secrets\pfx-cert.xml"

Option 3 - SecureString in script

$sec = Read-Host -AsSecureString -Prompt 'PFX password'
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew `
    -Scope PfxFile -PfxPath 'E:\cert.pfx' -PfxPassword $sec

Option 4 - Azure Key Vault

Connect-AzAccount
.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew `
    -Scope PfxFile -PfxPath 'E:\cert.pfx' `
    -KeyVaultName corp-kv-secrets `
    -KvSecretNamePfx 'ns2otpapp-pfx-password' `
    -KvSecretNamePse 'ns2otpapp-pse-pin'

Option 5 - Plaintext (lab only)

.\Update-ServerCert.ps1 -Target server01 -Credential $cred -Action Renew `
    -Scope PfxFile -PfxPath 'E:\cert.pfx' -PfxPassword 'p@ss'
Resolve order: provided -> saved DPAPI file -> Key Vault -> interactive prompt -> fail.

6. AZURE Azure VM targets

6.1 Target an Azure VM via WinRM (same as on-prem)

$cred = Get-Credential    # Azure admin credential
# Ensure NSG allows 5985/5986 from your jump host
# Ensure VM has WinRM enabled: Invoke-AzVMRunCommand ... -CommandId 'EnableRemotePS'
.\Update-ServerCert.ps1 -Target azvm01.contoso.internal -Credential $cred -Action Discover -Scope All
.\Invoke-AutoHeal.ps1 -Target azvm01.contoso.internal -Credential $cred -Mode Diagnose

6.2 Target via Azure RunCommand (no direct WinRM needed)

Connect-AzAccount
$script = Get-Content 'C:\Users\PC\Downloads\AutoHeal\Invoke-AutoHeal.ps1' -Raw
Invoke-AzVMRunCommand -ResourceGroupName rg-prod -VMName azvm01 `
    -CommandId 'RunPowerShellScript' -ScriptString $script `
    -Parameter @{ Target='localhost'; Mode='Diagnose' }

6.3 Store cert passwords in Azure Key Vault

Connect-AzAccount
$sec = Read-Host -AsSecureString -Prompt 'PFX password'
Set-AzKeyVaultSecret -VaultName corp-kv-secrets -Name 'azvm01-pfx-password' -SecretValue $sec

# Then:
.\Update-ServerCert.ps1 -Target azvm01 -Credential $cred -Action Renew `
    -Scope PfxFile -PfxPath 'C:\app\cert.pfx' `
    -KeyVaultName corp-kv-secrets -KvSecretNamePfx 'azvm01-pfx-password'

6.4 Azure Key Vault cert renewal

# List certs in the vault
.\Update-ServerCert.ps1 -Target localhost -Credential $cred -Action Discover -Scope AzureKv -KeyVaultName corp-kv-certs

# Import a freshly-signed PFX into Key Vault
.\Update-ServerCert.ps1 -Target localhost -Credential $cred -Action Renew -Scope AzureKv `
    -KeyVaultName corp-kv-certs -KvCertName 'app-tls' `
    -PfxPath 'C:\new.pfx' -PfxPassword 'p@ss'

6.5 Fleet-wide Azure VM sweep

Connect-AzAccount
$vms = (Get-AzVM | Where-Object { $_.OSProfile.WindowsConfiguration }).Name
.\Invoke-AutoHeal.ps1 -Target $vms -Credential $cred -Mode Heal -ApplyFixes `
    -Severity Critical,High -AutoApprove -NonInteractive

11. Scheduled / nightly runs

11.1 Nightly AutoHeal on jump server

$action = New-ScheduledTaskAction -Execute 'pwsh.exe' -Argument @'
-NoProfile -File "C:\Users\PC\Downloads\AutoHeal\Invoke-AutoHeal.ps1" -Target (Get-Content C:\ops\servers.txt) -Mode Heal -ApplyFixes -AutoApprove -NonInteractive -Severity Critical,High

                $HOME\.cert-secrets\pse-<name>.xml

13.2 Exit codes

CodeMeaning
0Success or Discover-only complete
1Terminating error (re-throw via $ErrorActionPreference='Stop')
Non-zero from -Action RenewAt least one renewal failed, but -Force was not set

13.3 AutoHeal finding object schema

@{
