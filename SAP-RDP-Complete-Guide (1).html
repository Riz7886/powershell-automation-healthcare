⚡ Serial Console bypasses EVERYTHING — no agent, no WinRM, no Bastion. Talks directly to the VM's COM port. This is your best shot right now.
Step 1 — Enable Serial Console in Azure Portal
Portal → VM → Help → Serial console
Storage Account prompt — Choose Managed Storage Account (simpler, no setup). Click Enable.
Wait 30–60 seconds for the console to initialize. You'll see a blank terminal or boot messages.
Press Enter once to wake it up. You should see a login prompt: Username:
Log in with local admin: Administrator or .\Administrator and your password.
Once logged in you get a C:\Windows\system32> CMD prompt. Paste the commands below.
Step 2 — Fix Winlogon via Serial Console CMD
Paste these one at a time — CMD prompt (not PowerShell)
Fix Shell Registry — CMD 
:: Fix Winlogon Shell
reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v Shell /t REG_SZ /d "explorer.exe" /f
Fix Userinit Registry — CMD 
:: Fix Winlogon Userinit (trailing comma is REQUIRED)
reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v Userinit /t REG_SZ /d "C:\Windows\system32\userinit.exe," /f
Enable RDP + Restart RDS — CMD 
:: Enable RDP
reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server" /v fDenyTSConnections /t REG_DWORD /d 0 /f

:: Open firewall for RDP
netsh advfirewall firewall set rule group="remote desktop" new enable=yes

:: Restart Remote Desktop Services
net stop "Remote Desktop Services" /y
net start "Remote Desktop Services"

:: Verify registry was set correctly
reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v Shell
reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v Userinit
Verify it Worked — CMD 
:: Shell should show: explorer.exe
reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v Shell

:: Userinit should show: C:\Windows\system32\userinit.exe,
reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v Userinit

:: RDP enabled check (should be 0x0)
reg query "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server" /v fDenyTSConnections
✅ After running those commands — try RDP immediately: mstsc /v:10.168.0.32 — NO reboot needed for the registry fix.

PowerShell Version — Use inside PSSession or Serial Console PS
Full Winlogon Fix — PowerShell 
$p = "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon"

# Check current values
Get-ItemProperty $p | Select Shell, Userinit

# Apply fix

Set-ItemProperty -Path $p -Name "Userinit" -Value "C:\Windows\system32\userinit.exe,"

# Enable RDP
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Terminal Server" `
                 -Name "fDenyTSConnections" -Value 0

# Open firewall

# Restart RDP service

Start-Sleep   4
Start-Service TermService

# Verify
Get-ItemProperty $p | Select Shell, Userinit
Write-Host "DONE - TEST RDP NOW" -ForegroundColor Green
