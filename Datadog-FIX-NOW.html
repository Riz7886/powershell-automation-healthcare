Part 1 - Mute the 36 No-Data monitors via Datadog UI

  1. Open https://us3.datadoghq.com/monitors/manage
  2. Apply filter: in the left sidebar, under Status, check only No Data (uncheck Alert/Warn/OK)
  3. The list shows all 36 No-Data monitors. Click the header checkbox at the top of the list to select all 36 at once
  4. In the toolbar above the list, click Mute
  5. Set duration to 7 days
  6. Paste this message into the note field: 
    Investigation - data ingestion broken upstream. Owner: Syed Rizvi. Auto-unmutes after 7 days.
  7. Click Mute to confirm
Result: All 36 monitors transition from No-Data to Muted. Dashboard goes clean. Auto-unmutes after 7 days; if data is still missing, monitor goes back to No-Data so it's not hidden forever.

Part 2 - Restart the Datadog Agent on the two MoveIT hosts

Open PowerShell as Administrator on your work laptop. Right-click PowerShell -> Run as Administrator. Paste these three lines, hit Enter after each.

2a. Allow WinRM to reach the two hosts (one-time setup)

Set-Item WSMan:\localhost\Client\TrustedHosts -Value "vm-moveit-auto,vm-moveit-xfr" -Force -Concatenate

2b. Restart agent on vm-moveit-auto

Invoke-Command -ComputerName vm-moveit-auto -ScriptBlock { Restart-Service datadogagent; (Get-Service datadogagent).Status }

2c. Restart agent on vm-moveit-xfr

Invoke-Command -ComputerName vm-moveit-xfr -ScriptBlock { Restart-Service datadogagent; (Get-Service datadogagent).Status }
Expected output for 2b and 2c: Running
If it errors with "credentials" or "access denied": add credentials to the call. Run this instead: 
$cred = Get-Credential
Invoke-Command -ComputerName vm-moveit-auto -Credential $cred -ScriptBlock { Restart-Service datadogagent }
Invoke-Command -ComputerName vm-moveit-xfr -Credential $cred -ScriptBlock { Restart-Service datadogagent }
Use your domain admin or local admin credential when prompted.
