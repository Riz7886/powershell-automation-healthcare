Overview. Two valid approaches to grant pipeline permissions in Azure DevOps: (A) grant a user direct access on a single pipeline, or (B) create a security group, add members, and grant the group access. This playbook gives you one script that does both. Use Scenario A for a one-off, Scenario B for anything recurring.
STEP 1
One-time prerequisites — install & log in

Run each command in an elevated PowerShell window. Skip any that are already installed.

1a — Install Azure CLI
winget install Microsoft.AzureCLI
If winget isn't available, download the MSI from https://aka.ms/installazcli.
1b — Add the azure-devops extension
az extension add --name azure-devops
1c — Log in with your PYX account
az login
A browser opens. Sign in with the account that has access to Pyx.DriversHealth. Close the tab when it says "You have logged in".
1d — Verify project access
az devops project show --org https://dev.azure.com/techmodgroup --project Pyx.DriversHealth --query "{name:name,id:id}"
Good: JSON blob with name and id. You're ready for Step 2.
Bad: "TF400813" or "not authorized" — your account doesn't have Reader on the project yet. Ask the DevOps admin to grant Reader, then retry.
STEP 2
Save the permissions script

Copy the full PowerShell below and save to:

C:\Users\<YOU>\Downloads\pyx-devops-perms.ps1
In Notepad: File → Save As → change "Save as type" to All Files → filename pyx-devops-perms.ps1 → Save.
STEP 3
Run the script

Open a new PowerShell window as Administrator and run:

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass cd $env:USERPROFILE\Downloads .\pyx-devops-perms.ps1
First line allows the script to run in this PowerShell session only — no system-wide change. Safe.

A menu appears. Keep the window open — every action loops back to the menu so you can do multiple things in one session.

Next time someone joins the team, re-run the script → option 2 to find the group → use the az devops security group membership add snippet below. No new pipeline grant needed.
Quick "add a member later" one-liner (outside the script)
az devops security group membership add --group-id <GROUP_DESCRIPTOR> --member-id newperson@pyxhealth.com --org https://dev.azure.com/techmodgroup
Get <GROUP_DESCRIPTOR> from menu option 2 — it's the long vssgp.Uy0xLTktMTU1... string.
TROUBLESHOOT
When things go sideways
ErrorFix
'az' is not recognizedAzure CLI not installed or PATH not refreshed. Close PowerShell, reopen as Admin, try again. If still failing, reinstall via winget install Microsoft.AzureCLI.
az login opens browser but never completesTry device-code flow: az login --use-device-code. Paste the shown code into the URL it gives you.
TF400813: Not authorizedYour account doesn't have access to the project. Ask the DevOps admin to add you as Reader on Pyx.DriversHealth.
The subject you provided is not validAzure DevOps can't find the user. (a) Verify the email spelling, (b) ask the user to log into the org at least once so they exist in the AAD directory, (c) try the display name instead.
Script can't be loaded because running scripts is disabledRun once in the same window: Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
Menu option 1 shows no pipelinesWrong project context. Re-run az devops configure --defaults organization=https://dev.azure.com/techmodgroup project=Pyx.DriversHealth and try again.
Permission applied but user still can't see pipelineAAD sync delay — takes up to 5 minutes. Ask user to fully sign out of Azure DevOps and sign back in.
